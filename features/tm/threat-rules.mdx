## Overview

Configurable Threat Generation Rules give project administrators full control over how threats are created.  
From realism tuning and reference formatting to evidence requirements and pattern customization, teams can adapt threat generation to match their frameworks, workflows, and compliance needs.

## Key Capabilities

<Columns cols={2}>
  <Card title="Custom Rule Patterns" icon="gear">
    Define custom generation patterns that shape how threats are structured, titled, and categorized.
  </Card>
  <Card title="Realism & Depth Settings" icon="sliders">
    Adjust the realism level to control the depth, detail, and contextual accuracy of generated threats.
  </Card>
  <Card title="Evidence Requirement Controls" icon="file-lines">
    Enforce mandatory evidence fields such as request samples, payload metadata, or replay artifacts.
  </Card>
  <Card title="CWE & Reference Formatting" icon="book">
    Configure how CWE IDs, OWASP categories, and external references should appear in generated outputs.
  </Card>
  <Card title="Reusable Rule Templates" icon="layer-group">
    Create rule presets that can be applied across multiple microservices, projects, or teams for consistent modeling.
  </Card>
  <Card title="Dynamic Adjustments" icon="arrows-rotate">
    Update rules at any time and instantly apply changes to future threat generation sessions.
  </Card>
</Columns>

## Value

Configurable Threat Generation Rules ensure threat outputs are aligned with your organization’s standards, compliance frameworks, and modeling expectations.  
This flexibility enables consistent, high-quality threat modeling — tailored to your engineering workflows and security maturity.
