## Overview

The **Subdomains** section in AIM provides a complete inventory of all discovered subdomains across your organization.  
It combines **discovery, enrichment, and risk visibility** into a single unified interface — allowing teams to track exposure, vulnerabilities, ownership, and infrastructure context in real time.

## Key Insights

### Vulnerability Distribution
Visual breakdown of vulnerabilities by severity, helping prioritize remediation.

### Environment Distribution
Shows how subdomains are spread across:
- Production  
- Staging  
- Unknown  

### Network Scope
Identifies whether subdomains are:
- External  
- Internal  

### WAF Coverage
Displays which subdomains are:
- Behind WAF  
- Not protected  

### Active vs Inactive
Distinguishes:
- Live assets  
- Dead or inactive endpoints  

### Status Code Distribution
Shows HTTP response patterns:
- 200 (Active)  
- 403 / 404 (Restricted or broken)  
- Other response anomalies

## Inventory Table

The inventory table is the **core operational view** where teams investigate and take action.

### Key Columns

| Column | Description |
|--------|-------------|
| **Asset Value** | Subdomain name with metadata (title / description if available) |
| **Status Code** | HTTP response status (e.g., 200, 403, 404) |
| **Vulnerabilities** | Number of associated vulnerabilities |
| **Reachability** | Whether the asset is externally reachable |
| **Environment** | Classification (Production, Staging, etc.) |
| **Owner** | Assigned user responsible for the asset |
| **Associations** | Linked assets (IPs, services, dependencies) |
| **Detected On** | First and last detection timestamps |

---

## Search and Filters

Powerful filtering enables precise asset discovery:

- **Is Vulnerable** — Filter assets with vulnerabilities  
- **Status Code** — Filter by HTTP response  
- **WAF Protection** — Identify protected vs exposed assets  
- **Externally Reachable** — Focus on public exposure  
- **Environment** — Filter by production, staging, etc.  

Filters can be combined to create highly targeted queries for investigation and triage.
