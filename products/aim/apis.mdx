## Overview

The **APIs** section in AIM provides a centralized inventory of all APIs discovered across your organization.  
It enables teams to track **endpoint exposure, vulnerabilities, ownership, and authentication posture** — helping identify risky or unmanaged APIs quickly.

## Dashboard View

Each API is backed by a dedicated **dashboard view** that provides high-level analytics before diving into individual APIs.

<Frame caption="API dashboard showing vulnerability distribution, auth insights, environment, and API exposure">
  <img src="/images/aim/apis2.png" alt="API Dashboard View" />
</Frame>

The dashboard highlights:

- **Total APIs** — Total number of APIs discovered  
- **No Auth APIs** — APIs without authentication (high-risk exposure)  
- **Vulnerable APIs** — APIs with associated vulnerabilities  
- **Critical APIs** — High-risk APIs requiring immediate action  

---

## Key Insights

### Vulnerability Distribution
Shows how vulnerabilities are distributed across APIs, helping prioritize remediation.

### Environment Distribution
Breakdown of APIs across environments:
- Production  
- Staging  
- Others  

### Internal vs External APIs
Identifies exposure scope:
- External APIs (public-facing)  
- Internal APIs  

### HTTP Method Distribution
Analyzes usage patterns across:
- GET  
- POST  
- PUT  
- DELETE  
- Other methods  

### Authentication Insights
Highlights APIs based on auth posture:
- Authenticated  
- No authentication (potential risk)  

### Collections by Workspace
Shows how APIs are grouped across workspaces or teams.

---

## Inventory Table

The inventory table is the **primary operational view** for managing APIs.

### Key Columns

| Column | Description |
|--------|-------------|
| **API Name** | Name of the API or service |
| **Endpoint Count** | Number of endpoints under the API |
| **Vulnerabilities** | Number of associated vulnerabilities |
| **Owner** | Assigned owner responsible for the API |
| **Detected On** | First and last detection timestamps |

---

## Search and Filters

Quickly locate and prioritize APIs using filters:

- **Is Vulnerable** — Show APIs with vulnerabilities  
- **Workspace** — Filter APIs by workspace or team  
- **Search** — Locate APIs by name  

These filters help isolate risky APIs, ownership gaps, or newly discovered services.

---

## Why This Matters

The APIs module helps transform API discovery into **actionable security intelligence**:

- Identify **unauthenticated APIs (critical exposure)**  
- Track **API sprawl and shadow APIs**  
- Prioritize **vulnerable APIs based on impact**  
- Enforce **ownership and accountability**  
- Understand **endpoint-level risk distribution**
