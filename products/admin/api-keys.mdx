# Developer Access

## API Keys

<Frame caption="API Keys entry point under Settings">
  ![API Keys card in Snapsec settings dashboard](/images/admin/api-keys-button.png)
</Frame>

## Overview

The **API Keys** module allows organizations to securely authenticate applications, adapters, and integrations with Snapsec.<br />API keys act as a **trusted access mechanism**, enabling programmatic communication between Snapsec services and external systems without exposing user credentials.

This module is essential for teams building integrations, automating workflows, or extending Snapsec’s capabilities across environments.

---

## Public API Key

<Frame caption="Public API Key view">
  ![Public API key displayed with copy and regenerate options](/images/admin/api-section.png)
</Frame>

Each workspace is issued a **Public API Key** that can be used to authenticate requests.

### Key Capabilities

- **Secure Authentication** — Verifies the identity of calling applications.
- **Easy Access** — Copy or regenerate the key directly from the UI.
- **Controlled Exposure** — Designed for programmatic use without sharing user credentials.

The API key acts as a gateway for all supported Snapsec integrations.

## What You Can Do With API Keys

<Frame caption="API key usage capabilities">
  ![API key capabilities such as adapters, secure routes, automation, and monitoring](/images/admin/second-api-section.png)

API keys unlock multiple integration and automation use cases:

### Connect Adapters

Link adapters across applications to enable seamless data exchange between Snapsec modules and external tools.

### Access Secure Routes

Authenticate requests to protected APIs, ensuring only authorized services can access sensitive endpoints.

### Automate Integrations

Trigger events, sync data, and orchestrate workflows across connected systems without manual intervention.

### Monitor Usage

Track API activity, request frequency, and integration performance to maintain visibility and security.

## Model Context Protocol (MCP)

<Frame caption="Developer Access navigation showing MCP Access">
  <img
    style={{ borderRadius: '0.5rem' }}
    src="/images/admin/mcp-1.png"
  />
</Frame>

The **Model Context Protocol (MCP)** enables AI development tools to securely communicate with Snapsec. By configuring an MCP server, developers can provide supported IDEs and AI assistants with controlled access to project context, security data, and Snapsec capabilities directly from their development environment.

MCP is managed centrally at the organization level, ensuring secure, standardized access across all connected clients.

## MCP Configuration

The **MCP Configuration** page allows administrators to enable or disable MCP access for the entire organization.

Administrators can:

- Enable or disable organization-wide MCP integration
- Control developer access to MCP-enabled clients
- Configure supported IDE connections
- Manage MCP settings from a centralized location

When enabled, developers can connect compatible IDEs using the provided configuration snippets.

---

## Supported IDEs & Clients

Snapsec provides ready-to-use MCP configuration for several AI-powered development environments.

- Codex CLI

Each client includes:

## IDE Configuration

Selecting an IDE displays a dedicated setup page containing everything required to establish a secure MCP connection.

Each configuration provides:

- Ready-to-copy configuration file
- Installation prerequisites
- Environment variable guidance
- IDE-specific setup instructions
- Secure API key recommendations

This allows developers to quickly integrate Snapsec into their preferred AI development environment without manual configuration.

<Callout icon="triangle-exclamation" color="amber">
  Never hard-code API keys into configuration files. Store credentials using environment variables or a secure secrets manager.
</Callout>

## MCP Sessions

The **MCP Sessions** page provides visibility into every active MCP connection established within the organization.

For each session, administrators can view:

- Connected IDE or client
- Device information
- Associated user
- Session status
- Vulnerability requests processed
- Session creation time

These insights help administrators monitor active AI integrations, audit usage, and identify connected development environments across the organization.
