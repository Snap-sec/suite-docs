## Overview

Groups help you:
- Segment assets by environment  
- Run focused vulnerability scans  
- Apply consistent policies across similar systems  
- Assign owners and teams to grouped assets  

Examples include:
- Production APIs  
- Staging environments  
- External applications  
- Cloud microservices

## Group Details

Each group contains:
- Assigned assets  
- Total vulnerability count  
- Severity distribution  
- Owners  
- Scan schedules  
- Group metadata and tags

## Group Actions

<Steps>
  <Step title="Create a Group" icon="plus">
    Add a new group with environment, risk, or service-based rules.
  </Step>
  <Step title="Add Assets" icon="folder-plus">
    Select assets from the catalogue to include in this group.
  </Step>
  <Step title="Assign Owners" icon="user-check">
    Allocate responsibility to teams or individuals.
  </Step>
  <Step title="Schedule Scans" icon="calendar-check">
    Enable recurring scans for the entire group.
  </Step>
</Steps>
