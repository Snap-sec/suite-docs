## Overview

<Frame caption="Change Management dashboard showing request insights and status overview">
  <img src="/images/vm/change-db.png" />
</Frame>

The **Change Management** module helps teams track and validate application or infrastructure changes before deployment.

It ensures that every release, feature update, or configuration change goes through a structured security review process.

<Callout icon="shield-check" color="green">
Every change request acts as a security checkpoint before deployment — ensuring controlled and auditable releases.
</Callout>

---

## What You Can Do

- **Create Change Requests**
  - Submit new requests for security validation  

- **Track Status**
  - Monitor requests as *Pending* or *Completed*  

- **Prioritize Work**
  - Assign priority levels like *Low* or *Medium*  

- **View Risk Impact**
  - Track vulnerabilities linked to each request  

<Callout icon="lightbulb" color="yellow">
Use priorities to align security testing with release urgency — critical fixes can be validated faster without delaying deployments.
</Callout>

## Change Requests List

<Frame caption="List view of all change requests with status and priority">
  <img src="/images/vm/change-requests.png" />
</Frame>

The **Change Requests** page provides a centralized view of all submitted requests.

### Key Details

- **Change Title**
  - Name of the request or release  

- **Vulnerabilities**
  - Linked findings (if any)  

- **Dates**
  - Request date and report deadline  

- **Priority**
  - Indicates urgency  

- **Assessment State**
  - Tracks progress (*Pending / Completed*)  

<Callout icon="eye" color="blue">
This view is ideal for quickly identifying pending validations and high-priority requests across teams.
</Callout>

Creating a request involves filling structured sections that capture all necessary context.

Providing complete and accurate details ensures faster validation and reduces back-and-forth during testing.
</Callout>

## Final Step

- Click **Create Request** to submit  
- The request appears in the Change Requests list  
- Status is automatically tracked  

<Callout icon="check-circle" color="green">
Once submitted, the request becomes part of your security workflow and can be tracked until completion.
</Callout>

## Why It Matters

Change Management ensures that every release is validated before going live.

It helps teams:
- Maintain visibility over all changes  
- Reduce risk from untested updates  
- Align development with structured security validation
