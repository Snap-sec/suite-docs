## Overview

The Normalization Engine allows you to **automatically classify and standardize vulnerabilities** using custom rules.

Instead of manually triaging findings, you can:

- Normalize inconsistent data from multiple sources  
- Apply severity or categorization rules  
- Reduce noise and false positives  
- Enforce consistent vulnerability handling

## Create Normalization Rule

<Frame caption="Create a new normalization rule">
  <img src="/images/vm/normalization-engine-x2.png" />
</Frame>

### Rule Configuration

Each rule consists of:

#### 1. Rule Details
- Rule Name  
- Description  

#### 2. Conditions
Define when the rule should apply:

- Field (e.g., Title, Severity, Source)  
- Operator (contains, equals, regex)  
- Value  

You can add multiple conditions for precise matching.

## How It Works

1. Define a rule using conditions  
2. System scans vulnerabilities  
3. Matches findings based on logic  
4. Applies normalization automatically

## Example Use Cases

- Tag all “Outdated Software” findings as High  
- Normalize scanner-specific naming differences  
- Reduce duplicate classifications  
- Automatically flag risky patterns
