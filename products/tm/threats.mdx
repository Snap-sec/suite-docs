## Search, Filters & Sorting

The Threat catalog includes powerful search and filtering:

| Filter | Description |
|--------|-------------|
| **Project** | Filter threats by project or microservice |
| **Severity** | Critical, High, Medium, Low |
| **State** | Pending â†’ Fix Verified |
| **Confidence** | Level of certainty in the finding |
| **Owner** | Assigned developer or analyst |
| **Endpoint** | Filter by API route or component |
| **Keywords** | Search within descriptions, references, or metadata |

## Threat Lifecycle Management

Threats move across a structured workflow:

<Steps>
  <Step title="Pending">
    Threat has been generated and awaits initial triage.
  </Step>
  <Step title="Acknowledged">
    Confirmed by analyst and assigned for remediation.
  </Step>
  <Step title="In Progress">
    Developer or team begins fixing the threat.
  </Step>
  <Step title="Fixed">
    Developer marks the threat as addressed.
  </Step>
  <Step title="Fix Verified">
    Security validates the fix and closes the threat permanently.
  </Step>
</Steps>
